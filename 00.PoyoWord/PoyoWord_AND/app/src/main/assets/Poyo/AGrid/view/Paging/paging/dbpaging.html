전체 데이터 조회후 그리드 기능을 이용한 Default paging 방식

Description

  • ● DB Paging을 사용하기 위해서는 SBGrid paging type : "page"을 필수로 설정 해주셔야 합니다.
  • ● 붉은 글씨체는 필수 속성이며, 사용방법은 다음과 같습니다.

Example

    
    SBGridProperties.paging = {
        type : 'page',		//Default Paging, gridajax Paging, jquery ajax Paging 공통적용
        count : 5,			//Default Paging, gridajax Paging, jquery ajax Paging 공통적용
        size : 5, 			//Default Paging, gridajax Paging, jquery ajax Paging 공통적용
        sorttype : 'all',		//Default Paging, gridajax Paging, jquery ajax Paging 공통적용
        showgoalpageui : true	//Default Paging, gridajax Paging, jquery ajax Paging 공통적용
    }

  •   1) type [필수][string]
  •    - DB페이징을 사용하실 경우 type을 "page" 로 설정이 필요합니다.
  •    - 전체 데이터 조회 후 그리드 기능의 paging을 사용하실 경우 type을 "all" 로 설정해 주셔야 합니다.
  •   2) count [필수][number]
  •    - 페이지 UI영역에 페이지 버튼의 갯수를 설정합니다.
  •    - ex ) 5개 설정시 [1][2][3][4][5] 와 같이 표시됩니다.
  •   3) size [필수][number]
  •    - 그리드 한화면에 표시될 행의 갯수를 설정합니다.
  •   4) sorttype [선택][string]
  •    - 그리드 페이징기능 사용시 헤더클릭시 정렬을 전체데이터 기준("all")으로 정렬할 것인지, 페이지 데이터 기준("page")으로 할 것인지 설정합니다.
  •    ※ gridajax 방식(gridajax Paging)을 통해서는 "all" or "page" 단위로 가능 하지만, jquery ajax 방식(jquery ajax Paging)을 통한 paging일 경우 "page" 속성만 사용이 가능합니다.
  •   5) showgoalpageui [선택][boolean]
  •    - 페이징 이동 UI를 설정합니다. (default : false)
